# タスク管理アプリ 仕様書

## 概要

毎日のタスクを効率的に管理するためのシングルページアプリケーション。
React + TypeScript + Tailwind CSS で実装されており、データはブラウザのローカルストレージに永続化される。

---

## 画面構成

### メイン画面

```
┌─────────────────────────────────────────┐
│          📝 タスク管理                    │
│      毎日のタスクを効率的に管理しましょう    │
├─────────────────────────────────────────┤
│  全タスク: N  |  進行中: N  |  完了: N   │
│              [ + タスクを追加 ]           │
├──────────┬──────────┬────────────────────┤
│  未対応  │  処理中  │     処理済み        │
│  (未着手)│ (進行中) │      (完了)         │
│          │          │                    │
│ [タスク] │ [タスク] │    [タスク]         │
└──────────┴──────────┴────────────────────┘
```

### タスク作成モーダル

「+ タスクを追加」ボタンをクリックするとオーバーレイで表示される。

## データモデル

### Task

| フィールド    | 型           | 必須 | 説明                          |
|-------------|-------------|------|-------------------------------|
| `id`        | `string`    | 必須 | 数値の連番。表示時にプレフィックスを付加 |
| `title`     | `string`    | 必須 | タスクのタイトル               |
| `description` | `string`  | 任意 | タスクの説明                   |
| `status`    | `TaskStatus`| 必須 | ステータス（下記参照）          |
| `workType`  | `WorkType`  | 必須 | 作業内容の種別（下記参照）      |
| `createdAt` | `string`    | 必須 | 作成日時（ISO 8601）           |
| `completedAt` | `string`  | 任意 | 完了日時（ISO 8601）。完了時に自動設定 |

### TaskStatus（ステータス）

| 値      | 表示カラム | バッジ色  |
|--------|-----------|---------|
| `未着手` | 未対応    | アンバー  |
| `進行中` | 処理中    | ブルー   |
| `完了`  | 処理済み  | エメラルド |

### WorkType（作業内容）

| 値         | 説明             |
|-----------|-----------------|
| `検証`    | 動作確認・検証作業 |
| `コード生成` | コードの実装・生成 |
| `環境整備` | 環境構築・設定作業 |

## 機能仕様

### 1. タスク一覧表示（ボードビュー）

- ステータスごとに3つのカラムに分けて表示
  - **未対応**（未着手）
  - **処理中**（進行中）
  - **処理済み**（完了）
- 各カラムにはタスク数バッジを表示
- タスクがない場合は「タスクはありません」と表示
- タスクがゼロの場合は下部に「まだタスクがありません」メッセージを表示

### 2. タスク統計

ボード上部のヘッダーに以下をリアルタイム表示：
- **全タスク数**（青）
- **進行中数**（アンバー）
- **完了数**（エメラルド）

### 3. タスク作成

**入力フィールド：**

| フィールド     | 必須 | デフォルト値    | 説明                        |
|-------------|------|----------------|----------------------------|
| IDプレフィックス | 任意 | `TASK-`       | タスクIDの前置文字列。永続化される |
| タスクタイトル  | 必須 | -             | 空欄では送信不可              |
| 説明          | 任意 | -             | タスクの詳細説明               |
| ステータス     | 必須 | `未着手`       | 作成時のステータス             |
| 作業内容      | 必須 | `コード生成`    | 作業種別                     |

**動作：**
- フォーム送信後、モーダルが自動で閉じる
- タスクは先頭に追加される（新しい順）
- IDはデフォルトで `TASK-1`, `TASK-2`... と採番される
- 次のID番号はフォーム内にプレビュー表示される

### 4. ステータス変更

- 各タスクカード内のセレクトボックスで直接変更可能
- `完了` に変更すると `completedAt` が自動設定される
- `完了` から他ステータスに戻すと `completedAt` はクリアされる
- 変更は即時反映・自動保存

### 5. タスク削除

- 各タスクカードのゴミ箱アイコンボタンで削除
- 確認ダイアログなしで即時削除

```

│ ID: TASK-1          [未着手] [コード生成] │
│                                       │
│ タスクタイトル（太字・大）              │
│                                       │
│ 説明テキスト（小・グレー）              │
│                                       │
│ 作成: 2026/03/29 12:54                │
│ 完了: （完了時のみ表示）               │
│                                       │
│ [ステータス選択 ▼]        [🗑️]        │
└──────────────────────────────────────┘
```

---

## IDプレフィックス仕様

- 初期値: `TASK-`
- 変更するとローカルストレージに保存され、次回起動時も反映
- 既存タスクの表示IDも即時更新される
- IDの数値部分は連番で、重複しないよう自動採番される
- データロード時にIDを `1` からの連番に正規化する

---

## データ永続化

| データ             | ストレージキー       |
|-------------------|-------------------|
| タスク一覧          | `tasks`           |
| IDプレフィックス    | `task_id_prefix`  |

- 保存形式: JSON
- タスク更新のたびに自動保存
- ページロード時に自動復元
